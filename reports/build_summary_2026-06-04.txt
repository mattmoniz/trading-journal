Trading Journal — Build Summary (June 4, 2026)

1. Off-Hours "Last Session" Fallback — All Dashboard Cards
----------------------------------------------------------

Problem: Four live-data cards in the dashboard (ACDView → DashboardCardGrid) showed blank or
error states when the market was closed because they only fetched real-time data.

Fix: Added a `lastSession` state that fires when `isRTH` (real-time hours) is false. It fetches
the most recent trade date from /api/acd/daily, then calls /api/case?date={lastDate}&asOf=16:00
to reconstruct end-of-session state.

Cards updated:
  - Trigger card  — shows last session's setup type, Entry/Stop/T1 levels, R:R, and state label
  - Levels card   — shows last session's OR levels, A Up/Down, POC, VWAP
  - Day card      — shows prior session's directional case (Bullish/Bearish/Neutral)
  - Case card     — shows last session's case-engine summary

Each card shows a LastSessionStamp italic footer: "last session · YYYY-MM-DD"


2. "Close to Forming" Amber Trigger Card
-----------------------------------------

Logic: When a setup is in WATCHING state (not yet active) and the structural confluence score
is >= 4/7, the trigger card turns amber.

Implementation:
  - isForming = !tActive && tState === 'WATCHING' && structScore >= 4
  - Card border becomes rgba(245,158,11,0.7), background gets a subtle amber tint,
    soft amber glow via box-shadow
  - Card header shows "forming · 4/7" (or whatever the current score is)
  - Returns to normal when score drops or setup fully activates


3. Session Timeline in Sidebar
-------------------------------

Replaced the static "Today's Signals" section with a live Session Timeline that merges
two event streams:

  - Level events from acd_setup_events (OR break, A Up/Down confirms, etc.)
  - Case-engine setups from /api/setups/today (active_setups table — TRT_LONG, TRT_SHORT, etc.)

Events are sorted by fired_time. Each case-engine setup shows:
  - Colored left border matching setup type
  - Entry, Stop, T1 prices
  - Outcome badge: T1 (green), Stop (red), expired (gray), or live (pulsing)
  - Reloads automatically on setup-resolved and setup-expired socket events

5. Combo Level Backtest System (scripts/combo_backtest.js)
-----------------------------------------------------------

New script (~530 lines) that computes level confluence stats from historical DB data.

What it does:
  - Fetches all NQ RTH sessions with ACD data from the DB
  - Computes 8 categories of key levels per session entirely from DB (no external data):
      ACD:              OR High/Low, OR5Mid, A Up, A Down
      Overnight:        ON High/Low
      Prior Day:        PD High/Low, VWAP
      Prior Week:       PW High/Low, W-VWAP
      PD volume profile: POC, VAH, VAL
      PM volume profile: PM VAL
      G-Line:           CME weekly open (Sunday 18:00 bar)
  - For each of 10 defined combos, checks whether all levels in the combo were within
    the proximity threshold (default 20 pts) of each other
  - Single-level combos (PM VAL) use touch detection (did session Hi or Lo reach the level?)
    rather than proximity
  - Outcome: NQ open→close × $20/pt — normalized, independent of which accounts were active
    (avoids sim/live contamination)
  - Upserts results to combo_stats table

Usage: node scripts/combo_backtest.js [--prox=20] [--dry-run]

8. Weekly Cron + Data Health Tracking
--------------------------------------

Weekly cron added to server/index.js:
  - Every Sunday at 6:30 PM ET, spawns combo_backtest.js as a detached child process

Data Health integration:
  - COMBO_BACKTEST entry added to PROCESS_SCHEDULE in server/routes/settings.js
  - combo_backtest.js writes its own process_log entries directly to the DB (necessary because
    it runs as a detached child — the parent server's logProcess() wrapper can't reach it):
      RUNNING  — inserted at script start
      SUCCESS  — updated with combo count on completion
      FAILED   — updated with error message if anything throws
  - The Data Health Status panel now shows a "Combo Level Backtest" row alongside Morning Brief,
    Auto-Import, Daily Coaching, etc. — gray until it runs, green after a successful Sunday run


Bug Fixes
----------

  - TDZ crash ("Cannot access 'isRTH' before initialization"):
      lastSession useEffect dependency array referenced isRTH before its const declaration.
      Fixed by moving the state and effect to after the isRTH declaration.

  - import { query, pool } SyntaxError:
      pool is the default export from db.js, not a named export.
      Fixed to: import pool from '../server/db.js'

  - PM VAL showing n=250:
      Single-level proximity check always passed (max-min of one value = 0).
      Fixed by adding touch-detection branch for single-level combos.

  - All combos showing negative avg P&L:
      Combined account P&L included sim/eval accounts with losses.
      Fixed by switching to NQ price-based outcome (open→close × $20/pt).

  - Morning brief "unavailable" error:
      Panel was defaulting to today's date even when no brief existed.
      Fixed to auto-select dates[0] (most recent) when today's brief isn't found.
